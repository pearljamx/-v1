# 驾驶注意力检测系统答辩演示脚本

适用时长：5 到 8 分钟

现场只需要快速启动时，可先查看 `docs/QUICK_START_CARD.md`。

## 1. 开场说明

本项目是“基于计算机视觉的驾驶注意力检测系统”。系统使用 Python、OpenCV、Flask、Bootstrap 和 YOLO 模型，支持图像、视频和摄像头三种输入方式，主要完成疲劳驾驶检测、分心驾驶检测和生理状态展示。

## 2. 启动项目

在项目目录中启动：

```bash
cd /Volumes/Data/02_课程/计算机视觉/实验/课程设计
source .venv/bin/activate
FLASK_PORT=5001 python app.py
```

浏览器打开：

```text
http://127.0.0.1:5001
```

说明要点：

- 后端使用 Flask。
- 前端使用 Bootstrap 风格页面。
- 计算机视觉模块按功能拆分在 `detectors/` 和 `processors/` 中。

## 3. 展示图像与视频检测

在首页展示上传入口：

1. 选择图片或视频文件。
2. 点击自动检测。
3. 展示检测结果和告警摘要。

讲解词：

系统会根据文件类型自动走图像或视频处理流程。图像处理由 `ImageProcessor` 完成，视频处理由 `VideoProcessor` 逐帧完成，检测结果统一返回到 Web 页面展示。

## 4. 打开实时摄像头页

打开：

```text
http://127.0.0.1:5001/camera
```

说明页面结构：

- 左侧或上方是驾驶舱实时画面。
- 页面绘制虚拟方向盘，用于判断手腕关键点是否处在握持区域。
- 指标面板展示人脸检测、疲劳指标、转头/转身、方向盘/手部、视线方向、光照、生理状态和活跃告警。
- 告警颜色分级：绿色为正常，黄色为警告，红色为危险，灰色为未知或等待数据。

## 5. 说明核心检测逻辑

疲劳检测：

- EAR 小于 0.2 且持续 3 秒以上，判断为闭眼疲劳。
- MAR 大于 0.5 且持续 3 秒以上，判断为哈欠。
- 眨眼频率低于 10 次/分钟时触发风险提示。
- 头部 pitch 超过 15° 持续 2 秒时判断为低头或点头。

分心检测：

- 视线偏离道路中心超过 30° 持续 2 秒触发提醒。
- YOLO 检测手机、吸烟、饮食等分心行为。
- YOLO pose 输出人体关键点，系统使用左右手腕关键点与虚拟方向盘 ROI 判断手离方向盘。
- 头部 yaw 超过 35° 持续 2 秒提示转头，超过 55° 时升级危险。
- 肩部/躯干扭转超过 45° 持续 2 秒提示转身取物。

生理状态：

- rPPG 模块选择前额 ROI，提取心率信号并展示血压趋势说明。
- 摄像头页默认关闭生理状态模块，避免影响实时帧率；需要展示时可手动开启。

## 6. 演示模式讲解

在摄像头页勾选“演示模式”。

讲解词：

现实阈值用于实际驾驶场景，避免短暂扫视或单手操作造成误报。课堂演示模式只缩短持续时间阈值，便于在有限时间内看到告警，不改变角度阈值、虚拟方向盘几何和关键点检测逻辑。

阈值对照：

| 检测项 | 默认现实阈值 | 演示模式阈值 |
| --- | --- | --- |
| 双手离把 | 5s danger | 1.5s danger |
| 单手离把 | 8s warning | 2s warning |
| 转头 | 2s warning/danger | 1s warning/danger |
| 转身 | 2s warning | 1s warning |

## 7. 无摄像头备用演示

如果现场没有摄像头或浏览器无法授权：

1. 在 `/camera` 页面展示“无摄像头备用演示”卡片。
2. 点击“固定样例”。
3. 展示系统从 `/api/demo/samples` 返回的样例图片列表。
4. 回到首页上传样例图，继续展示图像检测链路。

讲解词：

备用样例来自本地 `dataset/val/images` 等数据集目录。它不能替代实时摄像头演示，但可以证明项目在无摄像头环境下仍可展示检测流程和结果页面。

## 8. YOLO 训练证明

展示 README 中“外部数据集与 YOLO 训练”小节，说明：

- 推荐数据集是 Roboflow Universe 的 Driver fatigue and distraction，约 6.4k 图像，CC BY 4.0。
- Hugging Face `n7i5x9/driver-drowsiness-dataset` 用于驾驶疲劳分类训练证明。
- 当前可证明模型包括：
  - `models_data/yolo_driver_state.pt`
  - `models_data/yolo_drowsiness_cls.pt`

可运行命令：

```bash
python -m yolo.train_real_datasets --list
python -m yolo.train_real_datasets --dry-run
```

说明：

`--dry-run` 不下载数据、不训练，只检查训练脚本、数据配置、模型目录、已有模型文件和 ultralytics 依赖，适合答辩前快速证明训练入口完整。

## 9. 最终验收展示

运行：

```bash
python scripts/acceptance_check.py
```

说明验收脚本会检查：

- 项目关键文件。
- Python 依赖导入。
- OpenCV contrib。
- YOLO 模型文件。
- 摄像头实时演示增强。
- 真实数据集训练入口。
- Flask API。
- 单元测试。

预期结果：

```text
汇总: PASS=8 WARN=0 FAIL=0
```

## 10. 收尾说明

本系统完成了课程要求中的 OpenCV、YOLO、Flask、Bootstrap、模块化设计、图像/视频/摄像头处理、疲劳检测、分心检测和生理状态展示。当前环境唯一未完全验证的是真机摄像头动态效果，原因是本机没有可用摄像头；答辩前建议在有摄像头的机器上做一次完整人工演示确认。

## 11. 答辩前 5 分钟检查清单

答辩开始前按下面顺序快速确认：

- Chrome 浏览器已打开，并访问 `http://127.0.0.1:5001/camera`。
- 浏览器摄像头权限已允许；如浏览器弹出授权提示，选择允许。
- 摄像头页能正常显示实时预览，或能看到“无摄像头备用演示”卡片。
- 实时预览区域内能看到虚拟方向盘辅助线。
- “演示模式”开关可以正常切换。
- 关闭演示模式时，默认现实阈值仍用于实际检测。
- 打开演示模式时，持续时间阈值会缩短，便于课堂展示告警。
- 无摄像头时，“固定样例”按钮可以打开本地样例列表。
- 在终端运行 `python scripts/acceptance_check.py`，结果为 `PASS=8 WARN=0 FAIL=0`。
- 若摄像头动态效果无法现场验证，直接按第 7 节走无摄像头备用演示流程，并说明该风险来自硬件环境限制。
